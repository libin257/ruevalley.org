## 三、数据展示（1 个）：**Steam 玩家热度仪表板**

> **首页位置**：Hero 右侧或第二屏全宽卡片
> **宣传语**：*“Rue Valley 热度，实时跃动！”*
> **可视效果**：圆环仪表 + 24h 折线微图，颜色随好评率动态渐变。

* **一次性数据来源**：

  * 【SteamSpy API】`https://steamspy.com/api.php?request=appdetails&appid=2126190` ([steamspy.com][10])
* **JSON 示例（截断）**

```json
{
  "appid": 2126190,
  "name": "Rue Valley",
  "positive": 299,
  "negative": 161,
  "owners": "20,000 .. 50,000",
  "score_rank": 65,
  "price": "26.99"
}
```

* **处理逻辑**

  1. 构建 `fetch-steamspy.ts`，build 时请求并写入 `/public/data/steamspy.json`。
  2. 前端组件 `SteamGauge` 读取本地 JSON → 计算好评率 `(positive/(positive+negative))*100`。
  3. ECharts / Recharts 渲染仪表 & 好评率折线。
* **用户价值**：及时看到“评价涨跌”，激发讨论与分享。

## 四、轻量小功能（1 个）：**意向树快速查询器**

> **首页位置**：数据仪表板下方
> **宣传语**：*“输入关键词，一秒定位任务线！”*
> **交互**
>
> * **输入**：下拉选择「关键词」(角色/地点) 或搜索框。
> * **输出**：所需 Loop 编号、NPC、前置意向、奖励一览。
> * **效果**：点击“复制”复制攻略步骤。

### 数据获取与落地

1. **一次性抓取**：将【IntoIndieGames 全流程】页面转为结构化 JSON（字段：`intention`, `loop`, `npc`, `prerequisite`, `reward`）。([Into Indie Games][7])
2. 保存至 `/public/data/intentions.json`；文件 <200 KB，可随构建更新。
3. 组件 `IntentSearch` 在客户端读本地 JSON 进行前端过滤，无额外 API 调用。
4. 推荐用 `Fuse.js` 做模糊搜索，保证移动端毫秒级响应。

### 场景示范

* **场景**：玩家搜索“Fix Vending Machine”。
* **步骤**：【意向树快速查询器】→ 输入框键入关键词 → 实时匹配 → 输出「Loop 3 开始，先与 Robin 对话，再拨打 Ben 电话」等要点，点击“复制”即可黏贴到游戏中。数据源来自预下载的 `intentions.json`，无需二次网络请求。
